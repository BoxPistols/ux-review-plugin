# UX Review AI — Figma Plugin

Figma 上のデザインを AI でレビューするプラグインです。デザインの「完成度」— ビジネスゴール達成・エンジニア手戻り防止・視覚的一貫性 — を判定し、次にすべきアクションを具体的に提示します。

## 特徴

- **3ゴール構造のレビュー**: ビジネスロジック達成 / エンジニア手戻り防止 / UI品質
- **4つのレビューモード**: 情報設計 / ビジュアル / デザインシステム / 実装引継ぎ
- **構造チェック（静的チェック）**: 全モードで自動実行。AI 不要の高速チェック
- **UIコンポーネント自動識別**: レイアウト・色・接続線・opacity からパターンを推定
- **複数フレーム対応**: フレーム間の関係性（遷移/親子/状態違い）を分析
- **仕様根拠の自動添付**: SpecBridge API でWCAG/ARIA/MDNの根拠URLを追加
- **画像ペースト**: Cmd+V でスクリーンショットを貼り付け → マルチモーダルAI に送信
- **深掘りチャット**: レビュー結果に対して対話的に質問（コンテキスト維持、画像添付可）
- **指摘クリックでハイライト**: Figma 上の該当ノードを選択・ズーム
- **Markdown 出力**: 全文コピー / .md ダウンロード

## セットアップ

### 1. クローン

```bash
git clone https://github.com/BoxPistols/ux-review-plugin.git
cd ux-review-plugin
npm install  # テスト用
```

### 2. Figma で読み込み

1. Figma デスクトップアプリを開く
2. **Plugins → Development → Import plugin from manifest...**
3. `manifest.json` を選択

### 3. API キー設定

プラグイン右上の **⚙** → Model 選択 → API Key 入力 → **テスト**

| モデル | 取得先 |
|---|---|
| Gemini 2.5 Flash（無料） | [Google AI Studio](https://aistudio.google.com/apikey) |
| gpt-5.4-nano | [OpenAI Platform](https://platform.openai.com/api-keys) |
| gpt-5.4-mini | [OpenAI Platform](https://platform.openai.com/api-keys) |

## 使い方

### レビュー

1. Figma でフレームを選択（複数選択可）
2. ユーザーストーリーを入力（任意。入力すると精度向上）
3. レビューモードを選択
4. **Review** ボタンをクリック

### レビュー結果

- **パターン識別**: 画面に含まれるUIコンポーネントを自動識別して明記
- **スコア**: 基本85点、Critical -20、Warning -5、Good +3 で算出
- **3ゴール別の指摘**: 各指摘に「問題 → アクション → 参考URL」の構造
- **SPEC バッジ**: 構造チェックによる指摘（青）、AI 指摘（バッジなし）
- **仕様根拠リンク**: SpecBridge API による WCAG/ARIA/MDN の自動添付

### 深掘りチャット

- レビュー完了後に表示
- 選択ノードを変更しても前のレビューコンテキストを維持
- 質問の長さに応じて回答量を自動調整
- Cmd+V で画像添付可
- Cmd+Enter で送信 / Enter で改行

## レビューモード

| モード | フォーカス |
|---|---|
| **情報設計** | ユーザーフロー、情報階層、タスク完遂、状態設計 |
| **ビジュアル** | タイポグラフィ階層、カラー一貫性、スペーシング |
| **デザインシステム** | 命名規則、バリアント網羅性、トークン準拠 |
| **実装引継ぎ** | インタラクション明示、状態遷移、レスポンシブ、エッジケース |

全モードで構造チェック（静的チェック）が自動実行されます。

## 3ゴール構造

レビュー指摘は以下の3ゴールに分類されます。ゴール1が最優先:

| ゴール | 内容 | 例 |
|---|---|---|
| **1. ビジネスロジック** | ユーザーがタスクを完了できるか | フロー上の行き止まり、判断材料の不足 |
| **2. 手戻り防止** | エンジニアが確認なしで実装できるか | 状態未定義、インタラクション不明 |
| **3. UI品質** | 視覚的一貫性と操作性 | トークン非準拠、シグニファイア不足 |

## 構造チェック

AI を使わず、Figma のノード構造を直接分析する高速チェック:

### コンポーネント識別シグナル

| シグナル | 検出内容 |
|---|---|
| 構造の繰り返し | 同じ構造が3回以上繰り返される → パターン候補 |
| 接続線 | 子要素間の LINE/VECTOR → ステッパー/タイムライン候補 |
| 色差 | 子要素間の fill 色が異なる → 状態区別の存在 |
| opacity 差 | 半透明要素 → 無効化/完了状態 |
| clickable | インタラクション設定あり |

### インタラクション定義・状態・オーバーレイチェック

レイヤー名からインタラクティブ要素を検出し、ComponentSet のバリアント軸から状態の網羅性を検証。

## SpecBridge API 連携

レビュー結果の critical/warning 指摘に対して、[SpecBridge](https://a11y-spec-bridge.vercel.app) の仕様検索 API を呼び出し、WCAG/WAI-ARIA/MDN の根拠 URL を非同期で追加表示します。

- スコア 0.6 以上の結果のみ表示（精度重視）
- 2秒タイムアウト（API 障害時はスキップ）
- レビュー結果は先に表示、根拠は後から追加

## アーキテクチャ

```
ユーザーストーリー入力（任意）
    ↓
Review ボタン
    ↓
┌──────────────────────────────────────────────┐
│  Layer 1: 構造チェック（AI不要・即座・全モード）  │
│  レイアウト分析 + パターン検出 + 状態チェック     │
└──────────────┬───────────────────────────────┘
               ↓ 結果を AI プロンプトに注入
┌──────────────────────────────────────────────┐
│  Layer 2: AI レビュー                          │
│  3ゴール構造（ビジネス/手戻り/UI品質）で分析    │
│  画像添付時はマルチモーダル対応                   │
└──────────────┬───────────────────────────────┘
               ↓
┌──────────────────────────────────────────────┐
│  Layer 3: 仕様根拠（非同期・SpecBridge API）    │
│  WCAG/ARIA/MDN の根拠URLを後付け               │
└──────────────────────────────────────────────┘
               ↓
        マージして表示 + 深掘りチャット
```

## 技術仕様

- **JavaScript**: ES2017 互換（Figma ランタイム制約）
- **AI**: Gemini / OpenAI マルチモーダル対応
- **構造チェック**: レイアウト・色・opacity・接続線・Variants によるパターン検出
- **仕様根拠**: SpecBridge API（WCAG 2.2 / WAI-ARIA / MDN）
- **永続化**: Figma `clientStorage`
- **テスト**: vitest
