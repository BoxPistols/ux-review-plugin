# UX Review AI — Figma Plugin

Figma 上のデザインを AI でレビューするプラグインです。情報設計・ビジュアル・デザインシステム・実装引継ぎの4つの観点から分析し、デザイナーとエンジニア間の手戻りを削減します。

## 特徴

- **4つのレビュー観点**: 情報設計 / ビジュアル / デザインシステム / 実装引継ぎ
- **仕様チェック（静的チェック）**: 実装引継ぎ / DS モードで自動実行。AI 不要の高速チェック
- **指摘クリックでハイライト**: Figma 上の該当ノードを選択・ズーム
- **セベリティ別グルーピング**: CRITICAL / WARNING / GOOD
- **個別コピー**: 各指摘を個別にクリップボード
- **チャット**: レビュー結果について対話的に深掘り
- **レビュー履歴**: 自動保存、展開表示、個別削除（最大20件）
- **Markdown 出力**: 全文コピー / .md ダウンロード
- **ライト/ダークテーマ**: 切替対応

## セットアップ

### 1. クローン

```bash
git clone https://github.com/BoxPistols/ux-review-plugin.git
```

### 2. Figma で読み込み

1. Figma デスクトップアプリを開く
2. **Plugins → Development → Import plugin from manifest...**
3. `manifest.json` を選択

### 3. API キー設定

プラグイン右上の **⚙** → Model 選択 → API Key 入力 → **テスト**

| モデル | 取得先 |
|---|---|
| Gemini 2.5 Flash（無料） | [Google AI Studio](https://aistudio.google.com/apikey) |
| gpt-5.4-nano | [OpenAI Platform](https://platform.openai.com/api-keys) |
| gpt-5.4-mini | [OpenAI Platform](https://platform.openai.com/api-keys) |

## 使い方

### Review タブ

1. Figma でフレームを選択
2. レビュー観点を選択（情報設計 / ビジュアル / デザインシステム / 実装引継ぎ）
3. **Review** ボタンをクリック

**レビュー結果:**
- スコア（1-100）
- 現状分析 / 改善提案
- CRITICAL → WARNING → GOOD のグルーピング
- 各指摘に個別 Copy ボタン
- 指摘クリックで Figma 上の該当ノードをハイライト

**チャット:**
- レビュー完了後、下部にチャットUI が表示
- 「この指摘について詳しく」「エンジニアへの伝え方」等を対話的に質問

**出力:**
- **全文コピー**: Markdown 形式でクリップボード
- **MD ダウンロード**: `.md` ファイルとして保存

### Manage タブ

- Figma Variables の一覧表示
- プラグイン作成分 / 全体の削除
- Undo

## レビュー観点

| 観点 | フォーカス | 仕様チェック |
|---|---|---|
| **情報設計** | タスク完遂フロー、状態設計（空/エラー/ローディング）、意思決定支援 | — |
| **ビジュアル** | タイポグラフィ階層、カラー一貫性、スペーシングのリズム | — |
| **デザインシステム** | 命名規則、バリアント網羅性、ガイドライン記載品質 | 自動実行 |
| **実装引継ぎ** | インタラクション明示、API接続点、状態遷移図 | 自動実行 |

## 仕様チェック（静的チェック）

**実装引継ぎ** と **デザインシステム** モードでは、AI レビューに加えて仕様チェックが自動的に実行されます。このチェックは AI を使わず、Figma のノード構造を直接分析して即座に結果を返します。

情報設計 / ビジュアルモードでは実行されないため、通常のデザインレビューに影響しません。

### チェック内容

#### 1. インタラクション定義チェック

レイヤー名・コンポーネント名からインタラクティブ要素を検出し、デザイン上で定義すべき仕様が揃っているか確認します。

| 検出パターン | デザインで定義すべきこと |
|---|---|
| `modal` / `dialog` | タイトルまたはラベル、閉じるボタン |
| `tab` / `tabs` | 選択中タブの視覚的区別、各タブと表示コンテンツの対応 |
| `alert` / `toast` / `snackbar` | 自動で消えるか手動で閉じるかの定義、表示時間の指定 |
| `menu` / `dropdown` | 開閉状態の両方のデザイン、選択肢ごとのアイコン/ラベル定義 |
| `accordion` | 開閉状態の両方のデザイン、複数同時に開けるかどうかの定義 |

#### 2. 状態デザイン不足チェック

ComponentSet のバリアント軸を分析し、必要な状態デザインが欠けていないか検出します。

| コンポーネント種別 | 必要な状態 |
|---|---|
| ボタン系 (`button`, `btn`, `cta`) | default, hover, pressed, disabled, focused |
| 入力系 (`input`, `textfield`, `search`) | default, hover, focused, disabled, error, readonly, filled |
| チェックボックス系 (`checkbox`, `toggle`, `switch`) | checked, unchecked, indeterminate, disabled |
| セレクト系 (`select`, `picker`) | default, open, disabled, error |

#### 3. オーバーレイ振る舞いチェック

オーバーレイ要素（`modal`, `dialog`, `drawer`, `popover`, `overlay`, `sheet`, `sidebar`）を検出し、以下の振る舞い定義が必要であることを通知します。

- どこを押して開くか（トリガー要素）
- 閉じ方（閉じるボタン？ESCキー？背景クリック？）
- 背景の扱い（暗くする？操作不可にする？）

### 結果の表示

仕様チェックの結果は AI レビュー結果と同じリスト内に表示されます。

- **SPEC バッジ**: 仕様チェックによる指摘には青い `SPEC` バッジが付きます
- **AI 指摘**: 従来通りバッジなし
- **Markdown 出力**: `[SPEC]` / `[AI]` タグで区別されます

仕様チェック結果は AI プロンプトにも自動注入されるため、AI は基本的なスペック漏れの重複指摘を避け、タスク完遂フローや意思決定支援などの高度な分析に集中します。

### 設計思想

このプラグインの対象はノーコードのデザイナーです。仕様チェックのメッセージは WCAG 条項番号や ARIA 属性名ではなく、「デザインで何を定義すべきか」「未定義だとエンジニアがどう困るか」というデザイナーの責務に翻訳されています。

## アーキテクチャ

### 2層レビュー構成

```
Review ボタン
    ↓
┌─────────────────────────────────────────┐
│  Layer 1: 静的チェック（AI不要・即座）    │  ← handoff / system モードのみ
│  ノード名パターンマッチ + バリアント分析   │
└────────────────┬────────────────────────┘
                 ↓ 結果を AI プロンプトに注入
┌─────────────────────────────────────────┐
│  Layer 2: AI レビュー                    │  ← 全モード共通
│  タスク完遂フロー・意思決定支援の分析     │
└─────────────────────────────────────────┘
                 ↓
         マージして表示（SPEC + AI）
```

### 技術仕様

- **JavaScript**: ES2017 互換（Figma ランタイム制約）
- **AI**: Gemini / OpenAI 直接呼び出し
- **静的チェック**: JSON ルールセットによるパターンマッチ（AI 不要）
- **永続化**: Figma `clientStorage`
- **フォント**: Noto Sans JP
